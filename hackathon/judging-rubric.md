# Hackathon Judging Rubric

Score each team on a 1–5 scale per category. Total possible: 25 points.

---

## Category 1: Creativity & Usefulness (1–5)

| Score | Criteria |
|-------|----------|
| 5 | Innovative approach; the agentic experience clearly adds significant value to the app |
| 4 | Creative use of agents; adds noticeable value |
| 3 | Solid implementation of an obvious use case |
| 2 | Basic implementation with limited practical value |
| 1 | Minimal or off-topic implementation |

## Category 2: MAF Concepts Used (1–5)

| Score | Criteria |
|-------|----------|
| 5 | Uses 4+ MAF concepts correctly (agents, tools, sessions, workflows, middleware, agent-as-tool) |
| 4 | Uses 3 MAF concepts correctly |
| 3 | Uses 2 MAF concepts correctly (e.g., agent + tools) |
| 2 | Uses 1 MAF concept correctly (basic agent only) |
| 1 | Does not meaningfully use MAF concepts |

## Category 3: Working Demo (1–5)

| Score | Criteria |
|-------|----------|
| 5 | Flawless demo; handles multiple scenarios and edge cases |
| 4 | Working demo with minor issues; handles main scenario well |
| 3 | Demo works for the happy path; some rough edges |
| 2 | Partially working; significant issues during demo |
| 1 | Demo does not work or was not presented |

## Category 4: Integration Quality (1–5)

| Score | Criteria |
|-------|----------|
| 5 | Seamlessly integrated into the partner app; feels like a native feature |
| 4 | Well-integrated; clearly connects to existing app services |
| 3 | Connected to the app but feels like a bolt-on |
| 2 | Loosely connected; mostly standalone |
| 1 | Not integrated with the partner app |

## Category 5: Presentation & Teamwork (1–5)

| Score | Criteria |
|-------|----------|
| 5 | Clear, engaging presentation; all team members contributed; great storytelling |
| 4 | Good presentation; demonstrates team collaboration |
| 3 | Adequate presentation; covers the basics |
| 2 | Disorganized presentation; hard to follow |
| 1 | No presentation or minimal effort |
